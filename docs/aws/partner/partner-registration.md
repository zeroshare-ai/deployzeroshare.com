## Step 2: Select Partner Path

### 2.1 Partner Paths Available

| Path | Description | Best For |
|------|-------------|----------|
| **Software Path** | ISVs selling software on/for AWS | ZeroShare ✓ |
| Services Path | Consulting and implementation | SI/Consultants |
| Training Path | AWS training delivery | Training providers |
| Distribution Path | Software distribution | Distributors |

### 2.2 Software Path Requirements

**Tier 1: Registered (Entry Level)**
- Complete registration
- No additional requirements
- Access to Partner Central

**Tier 2: Validated**
- AWS Marketplace listing (live)
- Customer references (2)
- Technical validation
- Annual revenue threshold

**Tier 3: Differentiated**
- FTR completion
- Competency achievement
- Significant AWS usage
- Customer success stories

### 2.3 Software Path Enrollment

1. Log into Partner Central
2. Navigate to Programs → Partner Paths
3. Select "Software Path"
4. Complete enrollment form
5. Submit for review

## Step 3: AWS Marketplace Seller Registration

### 3.1 Seller Account Setup

**URL:** https://aws.amazon.com/marketplace/management/

**Prerequisites:**
- AWS account (separate from partner account)
- Tax information (W-9 for US)
- Bank account for disbursements
- Product ready for listing

### 3.2 Seller Profile

| Field | Value |
|-------|-------|
| Company Name | ZeroShare, Inc. |
| Display Name | ZeroShare |
| Logo | 500x500 PNG |
| Description | Enterprise AI Security Solutions |
| Website | https://zeroshare.io |
| Support Email | support@zeroshare.io |
| Support URL | https://support.zeroshare.io |

### 3.3 Banking Information

| Field | Required |
|-------|----------|
| Bank Name | Yes |
| Account Number | Yes |
| Routing Number | Yes |
| Account Type | Yes |
| Tax ID (EIN) | Yes |

- Complete W-9 (US) or W-8BEN-E (non-US)

- Maintain tax compliance documentation

### 4.2 Listing Submission

1. Navigate to Products → Create Product
2. Select product type (Container)
3. Complete product details
4. Configure pricing
5. Upload assets
6. Submit for review

### 4.3 AWS Review Process

| Stage | Duration | Action Required |
|-------|----------|-----------------|
| Submission | Day 0 | Submit listing |
| Initial Review | Days 1-3 | AWS reviews content |
| Technical Review | Days 3-7 | AWS tests product |
| Feedback | Days 7-10 | Address any issues |
| Approval | Days 10-14 | Listing goes live |

## Step 5: Foundational Technical Review (FTR)

### 5.1 FTR Overview

FTR validates your solution follows AWS best practices.

**Benefits:**
- "AWS Reviewed" badge
- Required for competencies
- Customer confidence
- Co-sell eligibility

### 5.2 FTR Requirements

See `docs/aws/technical/ftr-checklist.md` for detailed requirements.

**Summary:**
- Security best practices
- Well-Architected alignment
- Operational excellence
- Documentation completeness

### 5.3 FTR Process

| Step | Duration | Action |
|------|----------|--------|
| Self-Assessment | 2-4 weeks | Complete checklist |
| Submit | Day 0 | Submit via Partner Central |
| Review | 2-4 weeks | AWS reviews submission |
| Feedback | 1-2 weeks | Address any gaps |
| Approval | 1 week | Receive FTR approval |

### 5.4 FTR Submission Checklist

- [ ] Complete self-assessment questionnaire
- [ ] Prepare architecture documentation
- [ ] Prepare security documentation
- [ ] Prepare operational runbooks
- [ ] Submit via Partner Central
- [ ] Schedule review call (if requested)

| Benefit | Description |
|---------|-------------|

| Funding | POC and migration funding |
| Marketing | Joint marketing opportunities |
| Technical | AWS SA support |
| Business | Business development support |

### 6.2 Eligibility Requirements

| Requirement | Threshold |
|-------------|-----------|
| FTR Status | Completed |
| Marketplace Revenue | $100K+ ARR (or path to) |
| Customer References | 3+ AWS customers |
| Technical Readiness | FTR passed |

### 6.3 Application Process

1. Achieve FTR completion
2. Build customer references
3. Apply via Partner Central
4. Complete program requirements
5. Get assigned Partner Development Manager

## Step 7: AWS Security Competency

### 7.1 Competency Benefits

- Premium Marketplace placement
- "AWS Security Competency" badge
- Priority in AWS referrals
- Advanced co-sell opportunities
- Marketing support

### 7.2 Security Competency Categories

| Category | Description | ZeroShare Fit |
|----------|-------------|---------------|
| **Data Protection** | Protect data at rest/transit | ✓ Primary |
| Identity & Access | IAM and access control | ✓ Secondary |
| Infrastructure Protection | Network/host security | Partial |
| Threat Detection | Detect security threats | Partial |
| Compliance | Compliance automation | ✓ Supporting |

### 7.3 Requirements

| Requirement | Details |
|-------------|---------|
| FTR | Must be completed |
| Customer References | 5+ validated references |
| Case Studies | 2+ published case studies |
| Technical Validation | Deep-dive technical review |
| AWS Usage | Demonstrated AWS integration |

### 7.4 Application Process

1. Complete FTR
2. Build required customer references
3. Publish case studies
4. Submit competency application
5. Complete technical validation
6. Achieve competency status
